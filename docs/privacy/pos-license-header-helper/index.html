Privacy Policy for POS License Header Helper

Effective date: July 7, 2026

English

Overview

POS License Header Helper is a Chrome and Edge extension designed for internal KPOS deployments. The extension helps authorized POS, e-menu, and kiosk client devices add required client identity headers to KPOS API requests.

Single Purpose

The sole purpose of this extension is to support the client authorization workflow of an internal KPOS system. When the current tab matches a configured KPOS activation path, the extension adds X-Client-Sn and X-Client-Type headers only to requests whose path starts with /kpos/api.

Data Stored Locally

The extension stores the following configuration data locally in the browser extension storage:

  • Generated device ID.
  • Browser type.
  • Client serial number, also called SN.
  • KPOS activation path configuration.
  • Client type value.
  • Extension mode, such as client mode or operation mode.

This data is used only to complete the KPOS client authorization flow. It is not sold, shared with third parties, or used for advertising, analytics, or tracking.

Network Communication

The extension communicates only with the KPOS site currently visited by the user. It may call the following internal KPOS API endpoints:

  • /kpos/api/client/overview
  • /kpos/api/client/register
  • /kpos/api/client/unbind

The extension does not send data to the developer's own servers or to any third-party analytics, advertising, or tracking service.

Remote Code

The extension does not use remote code. All JavaScript, HTML, CSS, and configuration files are packaged with the extension. The extension does not download, load, or execute remote JavaScript, WebAssembly, plugins, or executable code.

Host Permission Scope

Internal KPOS systems may be deployed under different intranet domains, IP addresses, and ports. For that reason, the extension may request broad host access. In practice, the extension activates only when the current tab matches a configured KPOS path, and header injection is limited to /kpos/api requests in that tab. Static resources, normal pages, and iframe page loads do not receive the client authorization headers.

User Rights and Contact

Users can edit or remove extension configuration from the extension options page, or remove the extension from the browser to delete its local extension data. For privacy questions, contact the publisher using the contact email configured in the Chrome Web Store developer account.

中文

概述

POS License Header Helper 是面向公司内部 KPOS 系统的 Chrome 和 Edge 浏览器扩展。 本扩展用于帮助 POS、电子菜单和自助点餐终端完成 KPOS API 请求所需的客户端身份请求头注入。

单一用途

本扩展的唯一用途是支持公司内部 KPOS 系统的客户端授权流程。当当前标签页命中已配置的 KPOS 激活路径时，扩展仅为路径以 /kpos/api 开头的请求添加 X-Client-Sn 和 X-Client-Type 请求头。

本地存储的数据

扩展会在浏览器扩展本地存储中保存以下配置数据：

  • 本地生成的设备 ID。
  • 浏览器类型。
  • 客户端序列号，即 SN。
  • KPOS 激活路径配置。
  • 客户端 type 值。
  • 插件运行模式，例如客户端模式或运维模式。

这些数据仅用于完成 KPOS 客户端授权流程。扩展不会出售这些数据，不会与第三方共享这些数据， 也不会将这些数据用于广告、分析或追踪。

网络通信

扩展仅会与用户当前访问的 KPOS 站点交互，可能调用以下公司内部 KPOS API：

  • /kpos/api/client/overview
  • /kpos/api/client/register
  • /kpos/api/client/unbind

扩展不会将数据发送到开发者自有服务器，也不会将数据发送到任何第三方分析、广告或追踪服务。

远程代码

本扩展不使用远程代码。所有 JavaScript、HTML、CSS 和配置文件均随扩展包一起发布。 扩展不会下载、加载或执行远程 JavaScript、WebAssembly、插件代码或可执行代码。

主机权限范围

公司内部 KPOS 系统可能部署在不同的内网域名、IP 地址和端口下，因此扩展可能需要较宽的主机访问权限。 实际使用中，扩展只有在当前标签页命中已配置的 KPOS path 时才会生效，并且请求头注入范围限制为当前标签页内 /kpos/api 开头的请求。静态资源、普通页面和 iframe 页面本身不会被添加客户端授权请求头。

用户权利和联系方式

用户可以在扩展配置页修改或删除扩展配置，也可以从浏览器中移除扩展以删除本地扩展数据。 如有隐私相关问题，请通过 Chrome Web Store 开发者账号中配置的发布方联系邮箱联系发布方。
